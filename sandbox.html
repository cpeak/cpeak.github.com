Fixed vs Responsive spacing The original design spec for the 50/50 module calls for a specific distance between the headline to the top of the module, and the distance between the text and the bottom (the pink boxes). This creates a problem: locking the distance to a set px value means the module is not responsive and it does not adapt to different screen sizes.

The first example is the fixed value. At a normal window size, say on a smaller laptop, the height between the headline and the top of the browser is 105px. But as you pull the window open the module grows horizontally. Since the height is fixed the Doctor gets clipped at the top and the bottom. While this is correct to the spec provided in the Sketch file, the user experience is not acceptable.

The second example is responsive. At a normal window size the modules are close to the same size, however as you open the window up the module grows vertically to acommodate the width change preventing the Doctor from being clipped.
